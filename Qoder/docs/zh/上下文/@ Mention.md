# @ Mention

为 AI 对话添加上下文，让 AI 准确理解需求并生成符合项目规范的代码。支持添加代码文件、项目目录、本地附件和项目规则。

## 添加上下文的方式

| 方式        | 操作                   | 说明                                |
| --------- | -------------------- | --------------------------------- |
| **点击按钮**  | 点击输入框右下角的上下文选择按钮 `@` | 打开选择窗口，浏览所有类型                     |
| **输入 @**  | 输入 `@` 后继续键入         | 实时检索文件、目录、规则，高亮匹配字符               |
| **拖拽/粘贴** | 将文件或图片拖放到聊天框         | 支持 md、xmind、xlsx、docs、pdf、jpg、png |

## 上下文类型

| 类型               | 用途          | 典型场景                |
| ---------------- | ----------- | ------------------- |
| **@file**        | 引用代码文件或规则文件 | 查询实现、重构代码、理解模块      |
| **@folder**      | 引用整个目录      | 重构模块结构、生成测试、跨文件查询   |
| **@attachments** | 引用本地文件或图片   | 根据设计稿生成代码、分析文档、处理表格 |
| **@rule**        | 引用项目规则      | 按团队风格生成代码、统一规范      |

## @file

引用项目中的代码文件，支持查询和修改。

### 添加方式

* 输入 `@file` 后输入文件名搜索
* 从资源管理器拖拽文件到聊天框

> 规则文件（如 `.qoder/rules` 目录中的文件）也可通过 `@file` 搜索。

## @folder

引用整个目录，适合需要理解或修改一组相关文件的场景。

### 添加方式

输入 `@folder` 后输入目录名搜索

## @attachments

引用本地文件或图片，让 AI 分析文档内容、根据设计稿生成代码或处理表格数据。

### 支持的文件类型

| 类型   | 格式          | 场景               |
| ---- | ----------- | ---------------- |
| 图片   | jpg、png     | 生成前端代码、分析布局、提取文字 |
| 文档   | md、pdf、docs | 分析内容、提取信息、生成注释   |
| 表格   | xlsx        | 分析数据结构、生成处理代码    |
| 思维导图 | xmind       | 理解架构、生成代码框架      |

### 添加方式

* 输入 `@attachments` 选择本地文件
* 直接拖放或复制粘贴文件到聊天框
* 图片会在输入框顶部独立展示，支持预览

## @rule

引用项目规则，让 AI 遵循特定的编码风格、框架约定或业务规范。

规则会嵌入到每次模型调用的系统提示中，提供持久、可复用的上下文。适合定义编码风格、命名规范、框架使用约定等需要在整个项目中保持一致的指导信息。

### 添加方式

* 输入 `@rule` 选择已配置的规则
* 规则列表为空时，可通过快捷入口直接创建新规则，无需跳转到设置页面

> 规则文件（如 `.qoder/rules` 目录中的 `.md` 文件）既可作为 `@rule` 类型引用（持久上下文），也可通过 `@file` 类型搜索（普通文件）。

## 使用技巧

### 组合多种上下文

在同一个提问中组合多种上下文类型：

```
@src/api.js @tests/api.test.js @rule coding-style
为这个 API 模块补充测试用例，确保符合团队编码规范
```

### 利用推荐列表

输入 `@` 后，列表上方会推荐常用文件：

* **当前文件**：正在编辑的文件
* **最近打开**：最近访问的 2 个文件

点击推荐文件可快速添加，无需搜索。

### 实时检索

输入 `@` 后继续键入，Qoder 会实时检索匹配的文件、目录和规则，并高亮显示匹配字符。
