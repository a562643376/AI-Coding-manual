# 模型管理

你可以为企业配置全局生效的模型设置。

## 系统内置模型列表

TRAE 企业版提供的系统内置模型如下：

|  | **套餐类型** | **可用模型** |
| --- | --- | --- |
|  | 基础版 | Doubao-Seed-Code（默认模型） |
|  | 团队版 & 旗舰版 | Doubao-Seed-Code（默认模型）<br>GLM-5<br>GLM-4.7<br>MiniMax-M2.7<br>MiniMax-M2.5<br>Qwen3-Coder-Next<br>Kimi-K2.5<br>DeepSeek-V3.2<br>DeepSeek-V3.1-Terminus |

## 管理系统内置模型

你可以启用/禁用系统内置模型。默认模型无法被禁用。

1. 登录 [TRAE 企业版控制台](https://console.enterprise.trae.cn/)。

2. 前往 **企业配置** > **模型**。

3. 在 **系统内置模型** 部分，启用/禁用模型。

## 添加企业内置模型

> **提示**

> 仅支持旗舰版。

添加自定义模型并将其启用后，企业内所有成员均可在 TRAE IDE 内使用该模型。

使用以下步骤，为你的企业添加自定义模型：

1. 登录 [TRAE 企业版控制台](https://console.enterprise.trae.cn/)。

2. 前往 **企业配置** > **模型**。

3. 在 **企业内置模型** 部分右上角，点击 **添加模型** 按钮。

4. 在 **添加模型** 窗口中，完成以下操作：

1. 选择模型提供商。目前可选的供应商包括：AWS、Anthropic、OpenAI、Gemini、OpenRouter、Kimi-CN、Kimi-Global、DeepSeek、火山引擎、阿里云、腾讯云、模力方舟、硅基流动、PPIO、Novita、BytePlus 和自定义。

2. 若你选择的模型提供商为 **自定义**，选择具体的模型并填入 URL。

- 提示URL 需符合 OpenAI 的 Chat Completions 协议规范，并填写为 API 的基础路径（Base Path）。基础路径指的是接口的统一前缀，不包含具体的功能端点（如 `/chat/completions`）。例如，如果模型完整的请求 URL 是 `https://example.com/v1/ai/chat/completion`，那么你应填写的 URL 为 `https://example.com/v1/ai/`。

3. 填入模型的 API Key。

4. (可选) 在 **高级设置** 处，填入自定义请求地址。

- 提示若请求该模型提供商需要经过其他服务，可在此处填入自定义请求地址；若为空，系统将默认使用该模型提供商的官方请求地址。自定义请求地址同样需遵循上述对 URL 的要求。
- 若请求该模型提供商需要经过其他服务，可在此处填入自定义请求地址；若为空，系统将默认使用该模型提供商的官方请求地址。
- 自定义请求地址同样需遵循上述对 URL 的要求。

5. 点击 **保存** 按钮。

## 验证 Anthropic 模型的可用性

Anthropic 的模型不支持通过浏览器直接发起请求来验证可用性，你需要在本地终端使用 `curl` 来测试 API Key。使用以下脚本：

curl 'https://api.anthropic.com/v1/models' \-H 'x-api-key: {{你的 apikey}}' \-H 'anthropic-version: 2023-06-01'

## 设置默认模型

设置企业内成员所使用的默认模型。默认模型不支持禁用及删除。

1. 登录 [TRAE 企业版控制台](https://console.enterprise.trae.cn/)。

2. 前往 **企业配置** > **模型**。

3. 在 **默认模型** 部分，设置一个企业默认模型。

## 管理成员的自定义模型权限

为企业成员开启自定义模型权限后，他们可以在 TRAE IDE 内自行添加自定义模型并使用。

使用以下步骤，为你的企业成员开启自定义模型权限：

1. 登录 [TRAE 企业版控制台](https://console.enterprise.trae.cn/)。

2. 前往 **企业配置** > **模型**。

3. 在 **成员自定义模型** 部分，打开/关闭 **是否允许成员在 TRAE 客户端和插件添加自定义模型** 开关。

## 用量管理

你可以为系统内置模型和企业内置模型配置 Token 用量限额。详情参考配置 Token 用量限额。

## 查看用量

开通按量付费后，TRAE 会记录企业在权益外所消耗的、需额外付费的 Token 数。你可以在用量统计看板中查看模型总 Token 数、模型输入总 Token 数、模型输出总 Token 数。

1. 登录 [火山引擎 TRAE 控制台](https://console.volcengine.com/trae/overview)。

2. 从左侧导航栏中，选择 **用量统计**。

3. 按需设置筛选条件，包括模型、数据周期和时间维度。

- 看板中将显示对应数据。
